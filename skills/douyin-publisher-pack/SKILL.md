# Douyin Publisher Pack

## Output

Produce one file: `output/publish/pack.md`, containing every section below. Always give options for creative items — the user picks; never publish anything yourself.

### 1. Titles (5 options)

≤20 chars each. Mix these proven shapes (one each, then two free):

- 悬念问句: 床底下伸出一只手，他为什么不敢喊？
- 反差陈述: 救他命的人，正是要他命的人
- 数字/代价: 二十两银子，买走了他后半生
- 系列格式: 初刻拍案惊奇·程宰篇（上）｜...

For 上/中/下 series, titles must carry the episode marker and the same series prefix every episode.

### 2. Cover Text (3 options)

≤8 chars main line + optional ≤10 chars subtitle. Must be readable at thumbnail size; verbs and nouns, no full sentences.

### 3. Caption / 简介

2-4 lines: one-line hook (not repeating the title), one-line context (出自《初刻拍案惊奇》卷X), interaction prompt (猜结局/评论区聊), then hashtags.

### 4. Hashtags (5-8)

Mix: 2 broad (#民间故事 #历史故事), 2 format (#说书 #漫画解说), 1 series tag (consistent every episode), 1-2 episode-specific.

### 5. Risk Review

Check the script and visuals against common Douyin review triggers; flag, don't silently change:

- Violence/gore wording in title or cover (改用暗示: 没熬过去 vs 死了)
- 迷信 framing: present 因果报应 as 民间故事观点, avoid 宣扬 phrasing
- Era-inaccurate claims presented as history
- Music copyright (use platform library tracks at publish time)

### 6. Posting Checklist

- Cover frame uploaded (not auto-selected), title ≤ limit, episode marker correct
- 上集 published → pin a comment teasing 中集; schedule next episode while retention is hot
- Cross-check series collection (合集) assignment — every episode joins the series 合集
- Record publish time + first-hour data in `series-log.md` to learn best posting slots

## Series Conventions

Keep one `series-log.md` per IP: episode number, title used, publish time, views/retention notes. Read it before naming a new episode so conventions and learnings carry forward.
